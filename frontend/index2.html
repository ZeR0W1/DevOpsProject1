Service Overview

The application is split into three services so that each EC2 instance has a single, clear responsibility in the AWS architecture.

Frontend EC2

Public-facing nginx server that hosts the UI and routes requests to the backend.

  • Accessible from the browser
  • Hosts static assets
  • Can reverse-proxy API requests

Backend EC2

Runs the main Python application logic for validation and machine configuration handling.

  • Receives submitted machine data
  • Validates request payloads
  • Coordinates persistence and workflow

Worker EC2

Handles asynchronous processing so the backend remains responsive.

  • Processes background jobs
  • Generates summaries or reports
  • Supports future AWS integrations
